# CanLegal Constitution

## Core Principles

### I. Education Over Advocacy
All content must be educational, neutral, and policy-focused. No calls to action, no promotion of illegal use, and no advocacy framing. The UI must visually separate facts, analysis, and references from any interpretive commentary.

### II. Evidence, Traceability, and Data Integrity
Every legal status and policy statement must be backed by verifiable sources and update metadata. Data provenance, last-updated dates, and known gaps must be visible in context. Visualizations must never distort the underlying data.

### III. Accessibility, Inclusion, and Progressive Enhancement
WCAG-oriented design is mandatory (keyboard navigation, contrast, focus states, reduced motion support). Content and core navigation must remain usable without advanced client features. Dark mode is required and parity must be maintained across themes.

### IV. Performance-First Experience
Optimize for fast loading and smooth interaction on mobile and low-bandwidth networks. Large assets (maps, charts, media) must be lazy-loaded and never block critical content. Performance budgets apply to LCP/INP/CLS and bundle size.

### V. Consistent, Scalable Design System
UI/UX patterns, typography, spacing, and components must be reusable and consistent. Information architecture must scale to new countries, regions, and policy categories without redesign. Editorial polish is required across all sections.

## Quality, Security, and Privacy Standards

- Minimize data collection; no personal data storage by default. Analytics, if used, must be privacy-preserving and documented.
- Apply secure defaults: CSP, dependency hygiene, input sanitization for any external data, and safe rendering of rich content.
- Semantic HTML and SEO metadata are mandatory. Content must be indexable and structured for accessibility and search engines.
- Interactive data visualizations must include accessible alternatives and clear legends, labels, and status definitions.

## Development Workflow and Testing Standards

- Critical interactions require tests: search, filters, map selection, country details, theme toggle, and reduced-motion behavior.
- Review gates: accessibility checks, performance checks, data accuracy review, and visual consistency review.
- Documentation must describe data sources, update workflow, component usage, and content governance.

## Governance

- This constitution supersedes all other guidance for the project.
- Amendments require a written rationale, impact analysis, and migration plan.
- Violations must be documented with a remediation plan and deadline.
